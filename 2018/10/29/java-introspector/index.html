Java 内省和 BeanUtils

需求

  • 以后我们开发框架 的时候，我们是经常需要把一些数据封装到对象中的。
  • 根据配置文件的内容生产对象的对象并且要把对象的属性值封装到对象中。

    现在有一个类Person

和一个配置文件 obj.txt

introspector.Person
id=110
name=狗娃

根据配置文件的内容生产对象的对象并且要把对象的属性值封装到对象中。

采用反射的方法

采用内省的方法

通过Introspector类获得Bean对象的 BeanInfo，然后通过 BeanInfo 来获取属性的描述器（ PropertyDescriptor ），通过这个属性描述器就可以获取某个属性对应的 getter/setter 方法，然后通过反射机制来调用这些方法。

用内省其实也不方便，最好采用 BeanUtils

存在的问题： sun公司的内省API过于繁琐，所以Apache组织结合很多实际开发中的应用场景开发了一套简单、易用的API操作Bean的属性——BeanUtils。

  • BeanUtils主要解决 的问题： 把对象的属性数据封装 到对象中。
  • BeanUtils的好处：
    1. BeanUtils设置属性值的时候，如果属性是基本数据 类型，BeanUtils会自动帮我转换数据类型。
      1. BeanUtils设置属性值的时候底层也是依赖于get或者Set方法设置以及获取属性值的。
      2. BeanUtils设置属性值,如果设置的属性是其他的引用 类型数据，那么这时候必须要注册一个类型转换器。
